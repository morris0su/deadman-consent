üì± Application Overview

Deadman Alert is a life-safety iOS application that monitors user activity and sends emergency wellness check alerts to designated emergency contacts if the user shows no activity for 24+ hours.

This is NOT marketing or promotional messaging - it is an emergency-only notification system comparable to medical alert services (Life Alert, fall detection, etc.)

üìä Consent Requirements Summary

üéØ THREE EXPLICIT CALLS TO ACTION (CTAs):

  1. CTA #1: User manually enters emergency contact phone number (required field)
  2. CTA #2: User reads SMS disclosure, checks consent box, taps "I Understand and Agree" (mandatory)
  3. CTA #3: User reads Terms of Service (including SMS section), checks box, taps "Accept" (mandatory)

‚ùå Users cannot bypass any of these steps. Monitoring remains disabled until all consent actions are completed.

üîí Consent Verification & Logging

  • Timestamp Logging: All consent actions logged with Unix timestamps in AWS DynamoDB
  • Database Record: User ID, emergency contact phone, consent date/time, app version stored
  • Audit Trail: Complete consent history maintained for compliance
  • Cannot be bypassed: Backend API validates consent before allowing monitoring activation

üè• Comparable Services

Deadman Alert operates similarly to established emergency services that use SMS:
