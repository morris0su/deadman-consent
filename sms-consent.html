Deadman Alert - SMS Consent Documentation

Last Updated: November 11, 2025

Purpose: This document explains how Deadman Alert obtains user consent to send SMS emergency notifications to designated emergency contacts.

Overview

Deadman Alert is a life-safety iOS application that monitors user activity and sends emergency wellness check alerts to designated emergency contacts if the user shows no activity for 24+ hours. This is NOT marketing or promotional messaging - it is an emergency-only notification system.

SMS Consent Process

Users provide explicit, informed consent through a multi-step process within the Deadman Alert iOS application:

Step 1: Account Creation

User downloads the Deadman Alert iOS app from the App Store and creates an account using Sign in with Apple for secure authentication.

Step 2: Enable Safety Monitoring

User navigates to the safety monitoring settings within the app and taps "Enable Monitoring" to activate the incapacitation detection feature.

Step 3: Designate Emergency Contact

The app prompts the user to designate ONE emergency contact who will receive alerts. The user manually enters the emergency contact's phone number with format validation.

Step 4: SMS Disclosure

The app displays a prominent disclosure message:

"Your designated emergency contact will receive SMS text messages if you show no activity for 24 or more hours. This is an emergency-only notification system. Your contact can opt out at any time by replying STOP to any message."

Step 5: Explicit Confirmation

User must tap a button labeled "I Understand and Agree" to acknowledge the SMS disclosure and proceed. The user cannot enable monitoring without this explicit confirmation.

Step 6: Terms of Service

User reviews and accepts the complete Terms of Service, which includes detailed information about SMS emergency notifications, opt-out procedures, and privacy practices.

Step 7: Monitoring Activation

Only after completing ALL consent steps does the app begin monitoring and gain the ability to send emergency SMS notifications. All consent actions are logged with timestamps in our secure database for compliance purposes.

Emergency Contact Consent

Emergency contacts receive SMS notifications based on implicit consent through their relationship with the user:

  • The user explicitly designates them as an emergency contact
  • They are informed by the user that they will receive emergency alerts
  • This model is similar to medical alert systems (Life Alert, fall detection services, etc.)
  • Recipients can opt out at any time by replying STOP

Message Frequency & Volume

  • Frequency: Maximum 1 alert per user per 24-hour period
  • Volume: Less than 200 SMS messages per month across all users
  • Type: Emergency-only notifications (NOT marketing, promotional, or routine messages)
  • Recipients: Single designated emergency contact per user

Sample Messages

Emergency Alert Message

URGENT: Deadman Alert has not detected activity from John Smith's device for 24 hours. This may indicate an emergency. Please perform a wellness check immediately. Reply STOP to opt-out, HELP for info.

HELP Response (Automatic)

Deadman Alert: Emergency safety monitoring service. You receive alerts if a user shows no activity for 24+ hours. Reply STOP to unsubscribe, START to resubscribe. Support: support@deadmanalert.com

STOP Confirmation (Automatic)

You have been unsubscribed from Deadman Alert emergency notifications. You will not receive further messages. Reply START to resubscribe.

START Confirmation (Automatic)

You have been resubscribed to Deadman Alert emergency notifications. Reply STOP to unsubscribe, HELP for information.

Opt-Out Procedures

We provide multiple opt-out mechanisms to ensure full TCPA compliance:

  • SMS Keywords: Recipients can reply STOP, STOPALL, UNSUBSCRIBE, CANCEL, END, or QUIT to any message
  • Processing Time: Opt-out requests are processed immediately (within 1 second)
  • Confirmation: Automatic confirmation message sent upon opt-out
  • Opt-In: Recipients can opt back in by replying START, SUBSCRIBE, or UNSTOP
  • System Enforcement: Our system checks the opt-out list before every message send (no messages sent to opted-out numbers)

TCPA Compliance

Deadman Alert is fully compliant with the Telephone Consumer Protection Act (TCPA):

  • ✅ Explicit user consent obtained before any SMS sent
  • ✅ Clear disclosure of SMS purpose and frequency
  • ✅ Automatic STOP/START/HELP keyword processing
  • ✅ Opt-out requests honored immediately
  • ✅ Confirmation messages for all opt-out/opt-in actions
  • ✅ Emergency-only messaging (not promotional)
  • ✅ Extremely low volume (<200 messages/month)
  • ✅ Single recipient per user (not bulk/broadcast)
  • ✅ All consent actions logged with timestamps

Privacy & Data Protection

  • All user data encrypted at rest and in transit
  • Phone numbers stored securely in AWS DynamoDB
  • No sharing of user data with third parties
  • Users can delete their account and data at any time
  • Consent records retained for compliance purposes
  • GDPR and CCPA compliant data practices

Technical Implementation

Platform: iOS mobile application (Swift)

Backend: AWS Lambda, DynamoDB, API Gateway

SMS Service: Twilio

Opt-Out Management: Twilio managed opt-out list with automatic keyword processing

Security: HTTPS/TLS encryption, JWT authentication, HMAC request signatures

Use Case Justification

Deadman Alert serves a critical public interest by providing life-safety monitoring for:

  • Elderly individuals living alone
  • People with medical conditions at risk of sudden incapacitation
  • Remote workers in potentially hazardous environments
  • Anyone concerned about being unable to call for help in an emergency

The service is comparable to established emergency alert systems such as Life Alert, medical alert systems, fall detection services, and other life-safety monitoring solutions that use SMS for emergency notifications.
